Inpathy — Backend Spec

MVP v1.0 March 2026
Inpathy is a daily emotional check-in system designed to reduce friction and increase awareness between people who care about each other. This document defines all locked backend decisions for MVP implementation.
Auth + User
Authentication decisions
Q1 · Stay logged in after phone + OTP verification?
Determines session persistence · affects re-auth UX
YES
Q2 · Required fields at signup?
First name + phone number · minimum account object
YES — first name + phone
Q3 · Can users edit name and photo after signup?
First name · last name (optional) · profile photo · editable post-signup
YES
Check-in
Check-in decisions
Q4 · Can users edit or delete a check-in after submitting?
Once submitted — mood is locked · no edit · no delete
NO
Q5 · One check-in per ritual window max?
User can only submit once during 8–9PM window · no duplicates
ONCE
Q6 · User signs up during ritual window — does check-in count?
e.g. signs up at 8:15PM → baseline check-in counts as today's ritual
YES — counts as today's
Circle + Journey
Circle + journey decisions
Q7 · If user misses the window — does last mood stay visible?
Last mood persists on Circle screen · no blank state · status dot resets
YES — last mood stays
Q8 · What does the status dot mean?
Appears when user submits mood during 8–9PM window · resets each cycle
Checked in today 8–9PM
Notifications
Notification decisions
Q9 · Does 8PM reminder go to everyone or only unchecked users?
Send to all users · no filtering by check-in status
EVERYONE
Q10 · Notify users when someone in circle checks in?
In-circle check-in notification is MVP · build now
YES — MVP
API Endpoints
Derived from locked decisions

GET/auth/sessioncheck active session validity
POST/auth/signoutclear session token
GET/user/profilefetch user profile
POST/user/profileupdate name · photo
POST/checkinsubmit mood · say more · timestamp
GET/checkin/windowcheck if ritual window is active
GET/circlefetch members · latest mood · status dot
POST/circle/invitegenerate custom link · set 7-day expiry
POST/circle/joinauto-join · mutual add · after auth only
POST/circle/removemutual remove · both sides
GET/journey/:userIdfetch mood history · ordered newest first
POST/notifications/send8PM ritual reminder · all users
POST/notifications/checkinnotify circle when member checks in
